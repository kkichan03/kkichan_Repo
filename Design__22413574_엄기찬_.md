## 1. Introduction

### 1) Summary

지난 Analysis 문서에서 시스템이 **무엇을 하는가**에 맞춰 Use Case와 Domain Class를 분석했다. 본 보고서에서는 이전 보고서의 내용을 바탕으로 시스템의 기능들을 **어떻게 구현하는가**를 기술한 Design 보고서이다.

StockEasy는 초보 투자자가 복잡한 금융 정보를 한 페이지에서 쉽게 파악할 수 있도록, 국내 주요 지수·미국 선물 지수·원자재 시세·실시간 속보를 단일 웹 페이지에서 제공하는 시황 요약 서비스다. 본 문서에서는 이 시스템을 실제로 구현하기 위한 Class Diagram, Sequence Diagram, State Machine Diagram을 제시하고, 각 설계 요소에 대한 설명을 기술한다.

구현 언어는 Python(Flask)을 백엔드로, HTML/CSS/JavaScript를 프론트엔드로 사용한다. 데이터 수집은 외부 API 및 RSS 피드를 통해 이루어지며, 자동 갱신은 JavaScript의 `setInterval`을 활용한다.

### 2) Important Points of Design

- 각 데이터 조회 기능(국내 지수, 미국 선물, 원자재, 속보)은 독립적인 클래스로 분리하여 유지보수성을 확보한다.
- DataFetcher와 NewsCollector는 각 조회 클래스에서 공통으로 사용되므로 별도 클래스로 추출한다.
- 외부 API 호출 실패 시 기존 데이터를 유지하고 오류 메시지를 표시하는 예외 처리를 반드시 구현한다.
- 프론트엔드와 백엔드는 REST API 방식으로 통신하며, 각 엔드포인트는 JSON 형식으로 데이터를 반환한다.

### 각 클래스 설명

#### 1) MainController

| 항목 | 내용 |
|------|------|
| **역할** | 시스템의 진입점이자 중앙 제어 클래스. 웹 페이지 접속 시 각 조회 클래스를 초기화하고 렌더링을 조율한다. |

**(1) Methods**

- `+initPage(): void` : 페이지 최초 로드 시 모든 데이터 조회 클래스를 호출하여 초기 데이터를 화면에 렌더링한다.
- `+renderDomesticIndex(): void` : InquireDomesticIndex 클래스를 호출하여 국내 지수 영역을 갱신한다.
- `+renderUSFuturesIndex(): void` : InquireUSFuturesIndex 클래스를 호출하여 미국 선물 지수 영역을 갱신한다.
- `+renderCommodityPrice(): void` : InquireCommodityPrice 클래스를 호출하여 원자재 영역을 갱신한다.
- `+renderRealtimeNews(): void` : InquireRealtimeNews 클래스를 호출하여 속보 영역을 갱신한다.
- `+startAutoRefresh(interval: int): void` : AutoRefresher를 시작하여 설정된 주기마다 자동 갱신을 수행한다.
- `+stopAutoRefresh(): void` : AutoRefresher를 중지한다.

#### 6) DataFetcher

| 항목 | 내용 |
|------|------|
| **역할** | 외부 증권·원자재 API를 호출하고 응답 데이터를 파싱하는 공통 클래스. InquireDomesticIndex, InquireUSFuturesIndex, InquireCommodityPrice 에서 사용한다. |

**(1) Attributes**

- `-apiUrl: str` : 호출할 API의 기본 URL
- `-timeout: int` : API 호출 제한 시간(초)
- `-retryCount: int` : 실패 시 재시도 횟수

**(2) Methods**

- `+call(url: str, params: dict): dict` : 지정된 URL과 파라미터로 HTTP GET 요청을 보내고 JSON 응답을 반환한다.
- `+parse(raw: dict): dict` : API 원본 응답에서 필요한 필드만 추출하여 정제된 딕셔너리를 반환한다.
- `+handleError(): void` : API 호출 실패 시 오류 로그를 기록하고 빈 딕셔너리를 반환한다.

---

#### 7) NewsCollector

| 항목 | 내용 |
|------|------|
| **역할** | 뉴스 RSS 피드를 수집·파싱하는 클래스. InquireRealtimeNews에 데이터를 제공한다. |

**(1) Attributes**

- `-rssUrl: str` : 수집할 RSS 피드 URL
- `-timeout: int` : 피드 수집 제한 시간(초)
- `-retryCount: int` : 실패 시 재시도 횟수

**(2) Methods**

- `+fetch(url: str): list` : 지정된 RSS URL에서 피드 데이터를 수집하고 원본 항목 목록을 반환한다.
- `+parse(raw: str): list` : RSS XML 원본을 파싱하여 제목, 발행 시각, 링크를 포함한 딕셔너리 리스트로 변환한다.
- `+handleError(): void` : 수집 실패 시 오류 로그를 기록하고 빈 리스트를 반환한다.

---

#### 8) AutoRefresher

| 항목 | 내용 |
|------|------|
| **역할** | 설정된 갱신 주기(최소 10초 또는 매 분)에 따라 타이머를 실행하고, 주기 만료 시 모든 데이터를 자동으로 갱신한다. 프론트엔드의 JavaScript `setInterval`로 구현된다. |

**(1) Attributes**

- `-interval: int` : 갱신 주기(초 단위)
- `-timerId: int` : setInterval이 반환하는 타이머 ID
- `-isRunning: bool` : 자동 갱신 실행 여부

**(2) Methods**

- `+start(interval: int): void` : 지정한 주기로 타이머를 시작한다.
- `+stop(): void` : 현재 실행 중인 타이머를 중지한다.
- `+setInterval(sec: int): void` : 갱신 주기를 변경한다. 실행 중이면 재시작한다.
- `+onTick(): void` : 타이머 만료 시 호출되며, MainController의 각 render 메서드를 순서대로 호출한다.

## 3. Sequence Diagram

이 장에서는 Use Case별로 Sequence Diagram을 소개하고 설명한다. 백엔드(Flask)는 각 데이터 조회 클래스를 담당하며, 프론트엔드(JavaScript)는 MainController와 AutoRefresher 역할을 수행한다. 프론트엔드는 REST API(`/api/...`) 형식으로 백엔드에 데이터를 요청한다.

---

### 1) 국내 주요 지수 조회 (Use Case #1)

```
User          Browser(MainController)    Flask(InquireDomesticIndex)   DataFetcher   External API
 |                     |                           |                        |               |
 |---접속(URL 입력)---> |                           |                        |               |
 |                     |                           |                        |               |
 |                     |---initPage()-----------> |                        |               |
 |                     |                           |---fetch()-----------> |               |
 |                     |                           |                        |---call()---> |
 |                     |                           |                        |<--JSON data--|
 |                     |                           |                        |               |
 |                     |                           |   [API 성공]           |               |
 |                     |                           |<--parse(raw):dict---  |               |
 |                     |<--getFormattedData():dict-|                        |               |
 |                     |                           |                        |               |
 |<--화면 렌더링(KOSPI, KOSDAQ, KOSPI200 카드)----  |                        |               |
 |                     |                           |                        |               |
 |                     |   [API 실패]              |                        |               |
 |                     |                           |<--handleError()------  |               |
 |<--"데이터를 불러올 수 없습니다" 메시지 출력------ |                        |               |
```

사용자가 웹사이트에 접속하면 MainController의 `initPage()`가 호출된다. `InquireDomesticIndex.fetch()`는 DataFetcher를 통해 외부 API를 호출하고, 성공 시 정제된 데이터를 프론트엔드에 반환하여 KOSPI, KOSDAQ, KOSPI200 카드를 렌더링한다. API 호출이 실패하면 오류 메시지를 화면에 출력한다.

### 2) 미국 선물 지수 조회 (Use Case #2)

```

 |                     |                           |   [RSS 수집 실패]      |               |
 |                     |                           |<--handleError()------  |               |
 |<--"속보를 불러올 수 없습니다" 메시지 출력-------- |                        |               |
```

RSS 피드 수집이 성공하면 기사를 발행 시각 기준 내림차순으로 정렬하여 표시한다. 각 항목은 발행 시각과 헤드라인으로 구성되며, 클릭 시 원본 기사로 이동한다.

---

### 5) 데이터 자동 갱신 (Use Case #5)

```
Browser(AutoRefresher)    Browser(MainController)    Flask(각 조회 클래스)   External API/RSS
         |                          |                          |                    |
         |---start(interval)        |                          |                    |
         |                          |                          |                    |
         |   [타이머 만료]           |                          |                    |
         |---onTick()-------------> |                          |                    |
         |                          |---renderDomesticIndex()->|                    |
         |                          |                          |---call()---------> |
         |                          |                          |<---JSON data-------|
         |                          |<--getFormattedData()--   |                    |
         |                          |---renderUSFuturesIndex()->|                    |
         |                          |                          |---call()---------> |
         |                          |                          |<---JSON data-------|
         |                          |<--getFormattedData()--   |                    |
         |                          |---renderCommodityPrice()->|                    |
         |                          |                          |---call()---------> |
         |                          |                          |<---JSON data-------|
         |                          |<--getFormattedData()--   |                    |
         |                          |---renderRealtimeNews()--> |                    |
         |                          |                          |---fetch()--------> |
         |                          |                          |<---XML data--------|
         |                          |<--getLatestNews()-----   |                    |
         |                          |                          |                    |
         |   [화면 전체 갱신 완료]   |                          |                    |
         |---타이머 리셋             |                          |                    |
         |                          |                          |                    |
         |   [일부 API 실패]        |                          |                    |
         |                          |<--handleError() (실패 항목만 오류 표시)-------|
         |                          |   (성공 항목은 정상 갱신)  |                    |
```

AutoRefresher는 사용자가 설정한 주기(최소 10초 또는 매 분)마다 `onTick()`을 호출한다. `onTick()`은 MainController의 각 render 메서드를 순서대로 실행하여 모든 데이터를 갱신한다. 일부 API가 실패해도 성공한 항목은 정상적으로 갱신되며, 실패한 항목만 오류 메시지를 표시한다.
